Tian.Date

Sub namespace contains convenience functions for date-time manipulation.

Tian.Date.doy(date)

Static method

Parameters:
date - object, instance of Date

Returns: number

Returns day of year.

Tian.Date.format(date, mask)

Static method

Parameters:
date - object, instance of Date
mask - string, optional mask string

Returns: string

Returns a default formatted date-time string. If a mask is set, the format of mask will be used. Supported masks:

  • yy: short year, eg: 12
  • yyyy: long year, eg: 2012
  • m: short month, 0 - 12
  • mm: long month, 00 - 12
  • d: short day of month, 0 - 31
  • dd: long day of month, 00 - 31
  • h: short hour, 0 - 11
  • hh: 2-digit hour, 00-11
  • H: short hour, 0 - 23
  • HH: 2-digit hour, 00 - 23
  • M: short minute, 0 - 59
  • MM: 2-digit minute, 00 - 59
  • S: short second, 0 - 59
  • SS: 2-digit second, 00 - 59
  • l: 2-digit millisecond, 00 - 99
  • L: 3-digit millisecond, 000 - 999

Examples:

    Tian.Date.format(new Date()) // "Tue Mar 06 2012 10:24:55 GMT+0800 (CST)"
    Tian.Date.format(new Date(), "dd/mm/yy") // "06/03/12"
    Tian.Date.format(new Date(), "yyyy-mm-dd HH:MM:SS") // "2012-03-06 10:27:31"
    Tian.Date.format(new Date(), "yyyy-mm-dd HH:MM:SS.L") // "2012-03-06 10:28:22.464"

Tian.Date.parse(str)

Static method

Parameters:
str - string, A string representing the date

Returns: object

Generate a date object from a string. If the string could not be parsed, an invalid date is returned (i.e. isNaN(date.getTime())). The format for the string follows the profile of ISO 8601 for date and time on the Internet (see http://tools.ietf.org/html/rfc3339).

We don't call the native Date.parse because of inconsistency between implementations. In Chrome, calling Date.parse with a string that doesn't contain any indication of the timezone (e.g. "2011"), the date is interpreted in local time. On Firefox, the assumption is UTC.

Examples:

    Tian.Date.parse("2011")
    Tian.Date.parse("2011-10")
    Tian.Date.parse("2011-10-21")
    Tian.Date.parse("2011-10-21T20:00:01.123Z")
    Tian.Date.parse("2011-10-21T20:00:01.123+08")
